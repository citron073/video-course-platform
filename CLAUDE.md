# CLAUDE.md（動画講座プラットフォーム / プロジェクトルール）

> このファイルはこのリポジトリでの作業ルール。グローバル設定より優先される。
> AI運用の横断ナレッジは `~/.claude/knowledge/` を参照（特に `project_build_playbook.md` = 本プロジェクトで実証した型の正本）。

## プロジェクト概要

YouTube限定公開動画を埋め込んだ、1講座・複数レッスン構成の動画講座サイト。
技術: Next.js 16 (App Router) / Turso (libSQL) + Drizzle ORM / Tailwind v4 / Vercel。
設計の正本は [`docs/実装プラン.md`](./docs/実装プラン.md)。

## 🔴 テストの鉄則（最優先・厳守）

### テストを通すために、テスト仕様そのものを変更してはならない

- **禁止**: テストが落ちたときに、合格させる目的で**期待値（`expect`）・テストケース・アサーションを書き換える / 緩める / 削除する / `skip` する**こと。
- テストが落ちたら **まず実装側（プロダクションコード）のバグを疑い、実装を直す**。テストは仕様の固定点として扱う。
- テストを変更してよいのは次の場合**のみ**:
  1. **仕様が本当に変わった**（要件変更）。この場合は先に [`docs/実装プラン.md`](./docs/実装プラン.md) を更新し、変更理由を [`docs/進捗記録.md`](./docs/進捗記録.md) に1行残してからテストを直す。
  2. **テスト自体に明確な誤り**がある（誤ったセットアップ・タイポ等）。この場合も理由を `docs/進捗記録.md` に残す。
- いずれの場合も、**「テストが落ちたから期待値を実際の出力に合わせる」式の修正は禁止**。実装の正しさを先に検証する。

### 運用

- テストは `*.test.ts`。純粋ロジックは `lib/` に切り出してから `lib/*.test.ts` でテストする（DB/env非依存に保つ）。
- 実装を変えたら `npm test` を通してからコミット/完了とする。
- 詳細な手順は [`docs/テスト手順.md`](./docs/テスト手順.md)。

## 🔵 作業フロー（必須・着手前と完了後）

### 着手前: 必ずドキュメント（使用書・手順書）を読む
実装・テストに入る前に、関連する `docs/` を**先に確認してから**手を動かす。
- 設計・要件 → [`docs/実装プラン.md`](./docs/実装プラン.md)
- 操作・手順（seed/デプロイ等）→ [`docs/運用手順.md`](./docs/運用手順.md)
- テストの方針・書き方 → [`docs/テスト手順.md`](./docs/テスト手順.md)
- 今どこまで進んでいるか → [`Tasks.md`](./Tasks.md) ／ [`docs/進捗記録.md`](./docs/進捗記録.md)

手順書と違うやり方をしたくなったら、まず手順書側を更新してから実装する（実装と手順書を食い違わせない）。

### 完了後: 必ず `Tasks.md` を更新する
実装やテストが**1つ完了するたびに**、プロジェクト直下の [`Tasks.md`](./Tasks.md) を更新する。
- 完了した項目は `[ ]` → `[x]` にし、必要なら日付を添える。
- 新たに判明した残作業は `Tasks.md` に追記する。
- 詳細な経緯・教訓は [`docs/進捗記録.md`](./docs/進捗記録.md)、残作業の再開手順は [`docs/未実行タスク.md`](./docs/未実行タスク.md) に書き、`Tasks.md` はその要約・チェックリストとして保つ。
- `Tasks.md` の更新が済むまで「完了」と言わない。

## コードの置き場所（勝手に新場所を作らない）

| 種類 | 場所 |
|---|---|
| 画面・ルート | `app/`（App Router） |
| UIコンポーネント | `components/` |
| DBスキーマ/クライアント/クエリ/seed | `db/` |
| 純粋ロジック・設定 | `lib/`（DB非依存。テスト対象はここ） |
| 設計・記録・手順 | `docs/` |

- レッスンデータの正本は **`db/seed.ts`**。変更後は `npm run seed`（冪等）。
- 講座スラッグの単一情報源は `lib/config.ts` の `COURSE_SLUG`（seedの `COURSE.slug` と一致させる）。

## 作業の締め（Definition of Done）

1. **着手前に該当の `docs/`（使用書・手順書）を確認した**。
2. `npm run build` が成功する（型エラー0）。
3. `npm test` が green（テスト仕様は上記ルールに従い不変）。
4. **[`Tasks.md`](./Tasks.md) を更新した**（完了項目に `[x]`／新規残作業を追記）。
5. 変更内容を [`docs/進捗記録.md`](./docs/進捗記録.md) に1行記録。残作業の再開手順は [`docs/未実行タスク.md`](./docs/未実行タスク.md) へ。
6. 秘密情報（`.env.local` / Tursoトークン）はコミットしない。

---

## セキュリティ

- 認証なし（限定公開）。`app/layout.tsx` で `robots: noindex` を維持。
- 埋め込み動画は「公開」または「限定公開(unlisted)」のみ。private は不可。
- `TURSO_AUTH_TOKEN` 等の秘密は `.env.local` と Vercel 環境変数のみ。出力・コミットしない。
