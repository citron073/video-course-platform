# 運用手順

> よく使う作業の手順書。コマンドはプロジェクト直下で実行する。

## 日常の起動

```bash
npm run dev        # http://localhost:3000
```

## レッスンを追加・編集する

1. `db/seed.ts` の `SECTIONS` を編集

   ```ts
   {
     title: "第4章 応用",
     lessons: [
       { title: "応用テクニック", youtubeId: "実動画ID", description: "説明文" },
     ],
   }
   ```

2. 反映

   ```bash
   npm run seed     # 全削除→再投入（冪等）
   ```

3. ブラウザで確認（必要なら `npm run dev`）

> seed は毎回データを作り直す。手動でDBを編集していてもリセットされる点に注意。

## スキーマ（テーブル定義）を変えたとき

`db/schema.ts` を編集したら:

```bash
npm run db:push    # スキーマをDBへ反映
npm run seed       # データ再投入
```

DBの中身をGUIで見たいとき:

```bash
npm run db:studio  # Drizzle Studio
```

## YouTube動画ID

- `youtubeId` は URL の動画ID（11文字）。
  - `https://youtu.be/dQw4w9WgXcQ` → `dQw4w9WgXcQ`
  - `https://www.youtube.com/watch?v=dQw4w9WgXcQ` → `dQw4w9WgXcQ`
- 埋め込み可：**公開** / **限定公開(unlisted)**
- 埋め込み不可：**非公開(private)**

## Vercel へのデプロイ

### 1. Turso でDBを作る（CLIログインが必要・たにさん実行）

```bash
turso db create video-course
turso db show video-course --url        # → TURSO_DATABASE_URL
turso db tokens create video-course     # → TURSO_AUTH_TOKEN
```

### 2. 本番DBへスキーマ + データを入れる

ローカルの `.env.local` を本番値に書き換えてから:

```bash
npm run db:push
npm run seed
```

（終わったら `.env.local` をローカル用 `file:./local.db` に戻す）

### 3. Vercel 側の設定

- プロジェクトの Environment Variables に登録:
  - `TURSO_DATABASE_URL`
  - `TURSO_AUTH_TOKEN`
- デプロイ → ライブで動画再生まで確認。

## トラブル時のチェック

| 症状 | 確認 |
|---|---|
| `TURSO_DATABASE_URL 未設定` | `.env.local` があるか／`file:./local.db` 等が入っているか |
| 講座が「データがありません」表示 | `npm run seed` を実行したか |
| 動画が再生されない | 動画が private になっていないか／`youtubeId` が正しいか |
| 404 になる | レッスンIDが存在するか（seed後にIDが変わる点に注意） |
