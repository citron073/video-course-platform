# EP36：Vercelで公開する手順（提出用の公開URLを得る）

GitHub: https://github.com/citron073/video-course-platform （push済み）
DBは既にクラウド(Turso・東京)なので、Vercelに環境変数を入れれば動く。

## 0. 前提（済）
- [x] コードをGitHubにpush（Public）
- [x] DB=Turso クラウド稼働
- [ ] Vercelアカウント作成（↓ここから・本人の作業）

## 1. Vercelアカウント作成（本人）
1. https://vercel.com → **「Continue with GitHub」** で登録（GitHub `citron073` でサインアップ）
2. 無料の Hobby プランでOK

## 2. プロジェクトをインポート
1. Vercelダッシュボード → **Add New… → Project**
2. `citron073/video-course-platform` を **Import**
3. Framework は Next.js が自動検出される。Build設定はデフォルトでOK

## 3. 環境変数を設定（Deploy前に入れる）
「Environment Variables」に、手元の `.env.local` の値をコピーして入れる：

| キー | 値 | 備考 |
|---|---|---|
| `TURSO_DATABASE_URL` | `libsql://…turso.io` | .env.localの値 |
| `TURSO_AUTH_TOKEN` | （長い文字列） | .env.localの値 |
| `BETTER_AUTH_SECRET` | （64桁hex） | .env.localの値 |
| `GOOGLE_CLIENT_ID` | …apps.googleusercontent.com | .env.localの値 |
| `GOOGLE_CLIENT_SECRET` | GOCSPX-… | .env.localの値 |
| `BETTER_AUTH_URL` | （後で本番URLに更新） | 一旦 `https://video-course-platform.vercel.app` |
| `NEXT_PUBLIC_APP_URL` | 同上 | 同上 |

※ `BLOB_READ_WRITE_TOKEN` は任意（画像アップロードを使うときだけ）。

## 4. デプロイ → 本番URLを確認
- **Deploy** を押す → 数分で `https://video-course-platform.vercel.app`（または類似）が発行される
- このURLが提出物

## 5. 本番URLに合わせて2箇所を直す（重要）
デプロイで実URLが分かったら：
1. Vercel → Settings → Environment Variables で `BETTER_AUTH_URL` と `NEXT_PUBLIC_APP_URL` を**実URL**に更新 → **Redeploy**
2. **GCP（Google認証）**：OAuthクライアントの「承認済みのリダイレクトURI」に本番用を追加（完全一致）：
   ```
   https://video-course-platform.vercel.app/api/auth/callback/google
   ```
   ※ localhost用は残したまま、本番用を1行追加する。これが無いと本番のGoogleログインが `redirect_uri_mismatch` で失敗する。

## 6. 動作確認 → 提出
- 本番URLを開く → 講座一覧が出る（公開済み講座）
- ログイン → 管理画面 → 講座作成 が動く
- URLを提出フォームへ

## ⚠️ 公開前のセキュリティ注意（必ず対応）
- 練習中に作った **テスト管理者 `tani-cloud@example.com`（弱いパスワード）** がクラウドDBに残っている。
  **公開サイトでは危険**なので、次のどちらかを実施：
  1. 自分の**Googleアカウントでログイン**して、そのユーザーを admin にする（`npm run make-admin -- 自分のメール`）
  2. テストアカウントを削除する（管理画面 or DB操作）
- 秘密の値（トークン/シークレット）は Vercel の環境変数にだけ入れ、コードには絶対書かない。

## つまずき早見表
| 症状 | 対処 |
|---|---|
| ビルドは通るがGoogleログインが本番で失敗 | GCPに本番リダイレクトURIを追加（手順5-2）＋ `BETTER_AUTH_URL` を実URLに |
| 「500」やDB接続エラー | Vercelの `TURSO_DATABASE_URL` / `TURSO_AUTH_TOKEN` を確認 → Redeploy |
| 環境変数を変えても反映されない | 変更後は必ず **Redeploy**（ビルド時に読まれる） |
